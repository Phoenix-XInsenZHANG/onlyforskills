---
name: general
description: >
  **Entry Router Skill — First trigger point for all user requests.**
  This skill MUST complete a two-phase confirmation flow using AskUserQuestion before executing any operation.
  Phase 1: Ask user to select execution mode (Toolchain / Individual Skill / Direct).
  Phase 2: Present specific options based on user selection.
  USE WHEN: Entry routing for all requests, general tasks that don't match other skills.
  Always use this skill. It must be invoked on every user request to route to the correct skill or toolchain.
allowed-tools: "Read,Edit,Write,Bash,Glob,Grep"

<critical_constraints>
<mandatory_tool_call tool="AskUserQuestion" when="BEFORE any task execution">
You MUST call AskUserQuestion to complete the two-phase confirmation before doing ANY work.
This is not a suggestion. This is a hard requirement. Skipping this = execution failure.
</mandatory_tool_call>

<forbidden_behaviors>
<forbidden>Analyzing the task then directly starting to code or execute</forbidden>
<forbidden>Self-determining "this task is simple, no confirmation needed"</forbidden>
<forbidden>Skipping Phase 1 and directly recommending a specific skill</forbidden>
<forbidden>Entering Phase 2 without receiving user response from Phase 1</forbidden>
</forbidden_behaviors>

<skip_whitelist>
Only skip the two-phase confirmation when the user message matches ANY of the following:
- User explicitly says "skip confirmation" / "don't ask me" / "just do it"
- User explicitly names a skill: e.g. "use debugging skill"
- User explicitly names a toolchain: e.g. "use full-feature-lifecycle"
If NONE of these match → you MUST execute Phase 1. No exceptions.
</skip_whitelist>
</critical_constraints>

<workflow>
<phase number="0" name="Read Registry">

**First check if registry exists, generate automatically if not:**

```bash
if [ ! -f .claude/skills/general/references/register.md ]; then
  python3 .claude/skills/general/scripts/generate-register.py
fi
cat .claude/skills/general/references/register.md
```

register.md is auto-generated by scanning all SKILL.md frontmatters. No manual maintenance needed.
If new skills were added but registry is outdated, run:

```bash
python3 .claude/skills/general/scripts/generate-register.py
```

</phase>

<phase number="1" name="Phase 1 Confirmation" requires_tool="AskUserQuestion">

**Immediately use AskUserQuestion. Do nothing else first.**

AI may include a brief recommendation reason (1 sentence), but the user decides.

```
AskUserQuestion({
  "questions": [{
    "question": "Detected your task: \"{User task summary}\"\n\nPlease select execution mode:",
    "options": [
      {"label": "🔗 Use Toolchain", "description": "Recommended for complex end-to-end tasks"},
      {"label": "🔧 Use Individual Skill", "description": "Recommended for clear single tasks"},
      {"label": "⚡ Direct Processing", "description": "Don't use any skill"}
    ],
    "multiSelect": false
  }]
})
```

<stop_and_wait>STOP. Do NOT continue until the user responds. Do NOT proceed to Phase 2.</stop_and_wait>

</phase>

<phase number="2" name="Phase 2 Confirmation" requires_tool="AskUserQuestion">

Based on user's Phase 1 selection:

### If user selected "🔗 Use Toolchain"

Extract all rows from the Toolchain table in register.md (read in Phase 0).
Convert each row to an option, format: `{ID} ({Name}: {Included Skills})`

```
AskUserQuestion({
  "questions": [{
    "question": "Please select a toolchain:",
    "options": [ /* Dynamically generated from register.md toolchain table. NO hardcoding. */ ]
  }]
})
```

**All options MUST come from register.md. Zero hardcoded toolchain names.**

<stop_and_wait>STOP. Do NOT continue until the user responds.</stop_and_wait>

### If user selected "🔧 Use Individual Skill"

From register.md, match relevant skills based on user task keywords.
Select top 3-4 most relevant skills, plus general as fallback.
Each option format: `{ID} ({Name}: {most relevant description})`

```
AskUserQuestion({
  "questions": [{
    "question": "The following skills may be relevant to your task:",
    "options": [ /* Dynamically matched from register.md. NO hardcoding. */ ]
  }]
})
```

**All options MUST come from register.md. Zero hardcoded skill names.**

<stop_and_wait>STOP. Do NOT continue until the user responds.</stop_and_wait>

### If user selected "⚡ Direct Processing"

Skip Phase 2. Proceed directly to Phase 3 general mode.

</phase>

<phase number="3" name="Execute">

Based on user's final selection:

- **Selected toolchain** → Call each skill in the order defined by the toolchain
- **Selected individual skill** → Read the corresponding SKILL.md and follow its instructions
- **Selected direct processing / general** → Execute in general mode below

### General Mode

1. Understand the task
2. Formulate a plan (for complex tasks, explain plan before starting)
3. Execute
4. Verify results
5. Return results

Scope:
- General coding (write functions, refactor, explain code)
- Git operations (commit, branch, merge)
- File operations (create, move, rename)
- Q&A and explanations
- Any task not belonging to a specialized skill

</phase>
</workflow>

---

# Empty Response Protection

If user's response to AskUserQuestion is empty or unparseable:
1. Resend the same AskUserQuestion (max 2 retries)
2. After 2 retries with no valid response → default to "⚡ Direct Processing"

---

# Fallback: When register.md Generation or Read Fails

If script execution fails and register.md doesn't exist:

1. Try manual scan: `ls .claude/skills/*/SKILL.md` to get available skill list
2. Still execute Phase 1 confirmation, but only keep two options:
   - "🔧 Tell me the skill name you want (I'll look it up)"
   - "⚡ Direct Processing (no skill)"
3. If user provides a skill name, try reading `.claude/skills/{name}/SKILL.md`
4. Inform user to run `python3 .claude/skills/general/scripts/generate-register.py` to fix
