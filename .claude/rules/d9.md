# D9 Rules (Directus 9 Backend Extensions)

## Extension Structure
Each extension is self-contained under `derp-orq/<name>/`:
```
<name>/
├── package.json        # directus:extension.type = "endpoint"
├── tsconfig.json       # target ES2019, rootDir ./src
├── src/index.ts        # defineEndpoint((router, { services, getSchema, database, exceptions }) => { ... })
├── src/DBUtils.ts      # Slim copy per extension (NOT shared import)
└── dist/index.js       # Build output
```

## Build & Deploy
```bash
cd <name> && npx directus-extension build    # produces dist/index.js
cp dist/index.js ../extensions/endpoints/<name>/index.js   # deploy
```
`EXTENSIONS_AUTO_RELOAD=true` in `.env` — D9 hot-reloads without restart.

## Start D9
```bash
cd /Users/mac/projects/synque/orq/derp-orq && npx directus start   # port 8080
```

## Shared Types
Import from `../../shared/types/base`:
- `GenericContext`, `GenericResult`, `Request`, `Response`, `Query`, `Accountability`

## DBUtils Pattern
Each extension copies its own DBUtils. Key methods:
- `get_items_by_query(collection, query)` — wraps `ItemsService.readByQuery()`
- `get_items_by_query_generic<T>()` — typed version
- `update_item(collection, id, payload)` — wraps `ItemsService.updateOne()`

Constructor: `new DBUtils(context, req.accountability)` for authenticated, `new DBUtils(context)` for public.

## Response Format
All custom endpoints use: `{ status: boolean, message: string, data: T }`

## Relational Filtering
Deep filter through relations (no 2-step lookup needed):
```typescript
{ member: { code_other_1: { _eq: value } } }   // filters booking by member.code_other_1
```

## Collection Names
Singular: `order`, `booking`, `member`, `product`, `event`, `availability` (same as D11).

## Auth
- `req.accountability?.user` / `req.accountability.role` for auth check
- Static token for testing (in `.env.local`)
- Public endpoints: `new DBUtils(context)` without accountability
